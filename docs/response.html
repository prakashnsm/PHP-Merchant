When an operation is performed by a payment gateway, a response object is always returned. Note that each gateway might extend the base response class to provide additional functions.

The common API for gateway responses are described below. For additional response functions provided by individual gateways, you might need to check the gateway's documentation.

  • get()

    Returns a property of the response. Some payment gateways return the build version, or a special token ID in order to validate a payment. You'll need to refer to the gateway's documentation and use this get() function to retrieve the value of such properties.

  • is_successful()

    Returns true if the payment request is successfully sent, and the response from the payment gateway contains no errors.

    Note that this does not actually indicate the transaction status (whether the payment is successfully debited into the merchant account, or it is still pending). For that, you need to use is_payment_completed().

  • has_errors()

    Returns true if the response from the payment gateway contains errors.

    To actually get the error details, use get_errors()

  • get_errors()

    Returns an array containing errors. Each error is an array containing the following keys:

    • code: error code supplied by the payment gateway
    • message: error message
    • details: some payment gateways also supplied a "long error message" that is more verbose

  • get_response_string()

    Returns the raw response string from the payment gateway.
