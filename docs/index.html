What is PHP Merchant?

PHP Merchant is a PHP library that lets developers easily integrate multiple payment gateways into their PHP applications. This PHP library was inspired by active_merchant, a payment abstraction library written in Ruby.

Who is PHP Merchant for?

PHP Merchant is right for you if you want a library that:

  • provides a consistent programming interface across various payment gateway providers.
  • minimizes the amount of time to learn to use the API for various payment gateways.
  • minimizes the amount of code you need to write in order to process payments.
  • is being used in real projects and proven to be stable.
  • has clear and thorough documentation.

Why use PHP Merchant?

The goal of PHP Merchant is to provide a consistent API across various payment gateways, so that you won't have to deal with each gateway's quirks. For instance, with Paypal IPN, 'payment_status' could sometimes be 'Canceled_Reversal', but in Paypal Express Checkout, it's 'Canceled-Reversal'. These inconsistent details can be very confusing and it's hard to memorize them all. PHP Merchant helps by providing a consistent abstraction across all supported payment gateways to make your job faster and easier.

With PHP Merchant, payment gateway API are categorized into 7 types of functionality:

  • Purchase
  • Authorize
  • Capture
  • Void
  • Credit
  • Recurring
  • Card Store

There might be unique functionalities to some payment gateways that are also supported (for instance: Setup Purchase (SetExpressCheckout) for Paypal Express Checkout). There will be documentation for each payment gateway to point out those little details.
