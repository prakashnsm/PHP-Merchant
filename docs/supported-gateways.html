Basic Operations

Supported operations of payment gateways can be categorized into 7 types:

  • Purchase: to process a transaction. Authorization of fund transfer and capture is done simultaneously.
  • Authorize: to authorize a payment in advance. This is useful for delayed order fulfillment process.
  • Capture: to capture either a partial amount or the full authorization amount.
  • Void: to void a previous authorization.
  • Credit: to refund a customer for a transaction that was processed and successfully settled through the payment gateway.
  • Recurring: to make a recurring payment (e.g. monthly subscription).
  • Card Store: store the credit card (and/or other payment) information.

PHP Merchant provides an abstract API consistent with the above categorization, so that you won't have to worry about the deep technical details about how payment requests are made.
