PHP Merchant is an open-source project driven by community contribution. Your participation in this community project is more than welcomed! There are a few ways for you to do that.

Reporting Bugs and Issues

Another way you can contribute to PHP Merchant is by reporting any problems you find while using it. Simply create an issue ticket on our github issue tracker, and you'll get responses from other community members.

When creating a ticket, make sure you describe thoroughly the issue you're having. A good issue ticket addresses the following points:

  • Which payment gateway are you using?
  • What is the expected results?
  • What do you see instead?
  • Are there any PHP notices, warnings or error messages?
  • What version of PHP Merchant and PHP are you using?
  • Screenshots would be awesome.

Development

If you're using PHP Merchant, it's likely that you're already an experienced web developer. The community will benefit a lot from your expertise if you could contribute code to this project in the form of: patches, pull requests, new payment gateway development, and testing.

Coding Standards

PHP Merchant is a spin-off from a opensource plugin for WordPress. As a result, we're adopting WordPress coding standards.

For unit tests, the same coding standards apply. Also, the test case function name should be as specific as possible, even if it's long. The reason why length doesn't matter is because we don't have to call unit test functions anywhere else, and we need to know which each test case does at the first glance when something fails.

Patches

Sending us a patch (via the issue tracker) would be the most straight-forward way for you to contribute code to the project. Let's say you have made some changes that you think would be useful for other users, you can create a patch on your local working copy:

{% highlight bash linenos %} # confirm the changes you made git status # export your changes to a patch file git diff > ~/Desktop/my-patch.diff {% endhighlight %}

Then upload your diff file so that other members can review and comment your change.

Pull Requests

Patches might be convenient for small changes (bug fixes, minor improvements). But if you want to contribute new features or new payment gateways, the best way would be to use Pull Requests. Github has a very detailed guide on how to use pull requests for collaborative development. Give it a read!

New Payment Gateways

If you've already extended PHP Merchant to support gateways that are not listed on our Supported Gateways page, why not share it with other developers? PHP Merchant is a new project and desparately needs to expand its support for as many payment gateways as possible.

You're recommended to take a look at our General Topics and Advanced Topics in the Table of Contents to know how to extend PHP Merchant feature.

Testing and Unit Tests

For production site, you're recommended to use the master branch. But if you want to develop using cutting edged versions, feel free to checkout code from our develop branch.

Included in PHP Merchant is a test suite, located in the tests sub-folder. We're trying to ensure every class has a good unit test coverage. It would be awesome if you could contribute edge test cases, or improve the test suite's code.

Documentation

The main source of documentation for this project is located in our gh-pages branch on github. The static content of the documentation is managed using jekyll. jekyll's documentation is pretty straight forward, and you can just look at our documentation structure to see how it works.

As the project evolves and the number of supported gateways grows, contribution from volunteers would help make PHP Merchant more accessible to new developers.
